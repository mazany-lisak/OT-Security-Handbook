# OT Security Handbook

An open, AI-ready knowledge base for **Operational Technology (OT)**, **Industrial Control Systems (ICS)** and **Industrial Cybersecurity Engineering**.

---

## Vision

The OT Security Handbook aims to provide a structured, vendor-neutral and engineering-focused knowledge base for designing, operating and securing industrial systems.

Unlike traditional documentation, this handbook is designed for both:

* Human engineers
* AI assistants (LLMs, RAG, Knowledge Graphs)

The goal is not to summarize standards, but to explain **how experienced OT architects think**.

---

## Project Goals

* Build a high-quality OT cybersecurity knowledge base.
* Use vendor-neutral engineering principles whenever possible.
* Base recommendations on official standards and authoritative sources.
* Optimize documents for AI-assisted reasoning.
* Support future Retrieval-Augmented Generation (RAG) systems.
* Remain technology-independent and easy to maintain.

---

## Scope

The handbook covers topics including:

* Operational Technology (OT)
* Industrial Control Systems (ICS)
* Industrial Automation and Control Systems (IACS)
* Cybersecurity Architecture
* Risk Management
* Industrial Networking
* Identity and Access Management
* Secure Remote Access
* Threat Intelligence
* Industrial Standards
* Industrial Protocols
* Vendor Documentation
* Security Best Practices

---

## Design Principles

The handbook follows several key principles:

* Architecture before technology.
* Risk-based security.
* Safety before cybersecurity.
* Explain reasoning rather than only recommendations.
* Prefer official terminology.
* Avoid vendor lock-in.
* Keep documents modular.
* Optimize for both humans and AI.

---

## Intended Audience

Primary audience:

* OT Security Architects
* Automation Engineers

* Security Consultants

Secondary audience:

* Students

* AI assistants
