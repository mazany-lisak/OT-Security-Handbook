# Contributing

Thank you for contributing to the OT Security Handbook.

The goal of this project is to create a high-quality, AI-ready engineering knowledge base for Operational Technology cybersecurity.

---

# Guiding Principles

Contributions should:

* Improve technical accuracy.
* Improve clarity.
* Remain vendor-neutral whenever possible.
* Reference authoritative sources.
* Explain architectural reasoning.
* Avoid unnecessary duplication.

---

# Preferred Sources

Always prefer official documentation.

Priority:

1. IEC
2. ISA
3. NÚKIB
4. ENISA
5. CISA
6. MITRE
7. NIST
8. Official vendor documentation

Blogs and marketing materials should only be used as supplementary references.

---

# Writing Style

Follow the rules defined in `STYLE_GUIDE.md`.

Documents should:

* Explain concepts.
* Explain reasoning.
* Explain trade-offs.
* Link related topics.
* Include AI Guidance where appropriate.

---

# Versioning

Use semantic versioning:

Major.Minor.Patch

Examples:

* 1.0.0
* 1.1.0
* 1.2.3

---

# Commit Messages

Recommended format:

```text
docs(category): short description
```

Examples:

```text
docs(legislation): add NIS2 overview
docs(standards): update IEC 62443 references
docs(vendors): add Siemens S7-1500 architecture
```

---

# Review Checklist

Before submitting:

* Technical accuracy verified
* Terminology consistent
* References reviewed
* Related documents linked
* Markdown formatting validated

---

# AI Considerations

Remember that these documents are consumed by both humans and AI.

Prefer:

* Short sections
* Descriptive headings
* Explicit relationships
* Consistent terminology
* Minimal duplication

---

# Definition of Done

A document is considered complete when:

* Purpose is clearly defined.
* Scope is complete.
* Technical content is accurate.
* Architecture context is included.
* References are provided.
* AI Guidance is included when applicable.

Quality is preferred over quantity.
